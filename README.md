# Learning Designer JD Matcher

This is a tiny local web app that helps you paste in a job description and quickly spot the parts that clearly fit the “Shane Legg laptop rule” idea:

- screen work
- keyboard work
- mouse work
- microphone work
- AI-friendly tasks

It also highlights phrases that are useful for learning design work, like course design, faculty advising, media recommendations, AI use, and project coordination.

Made with Codex from OpenAI: [openai.com](https://openai.com)

## What You Need

You only need:

- a Windows computer, Mac, or Linux machine
- a web browser such as Chrome, Edge, Safari, or Firefox
- this folder on your computer

You do not need to install any software to use the app. You can open it directly in your browser.

## What Is In This Folder

You should see these files:

- `index.html`
- `styles.css`
- `app.js`
- `README.md`
- `.gitignore`

## How To Open The App

1. Open the `learning-designer` folder.
2. Find the file called `index.html`.
3. Double-click `index.html`.
4. Your browser should open and show the app.

If nothing happens when you double-click it:

1. Right-click `index.html`.
2. Choose **Open with**.
3. Pick your web browser.

## How To Use The App

1. Paste a job description into the big text box.
2. Click **Analyse text**.
3. Read the highlighted matches under **What the app found**.
4. If you want a fresh start, click **Clear**.
5. If you want an example first, click **Load sample text**.

You can also press:

- `Ctrl + Enter` on Windows and Linux
- `Cmd + Enter` on Mac

to run the analysis without clicking the button.

## What The App Is Looking For

The app looks for language that suggests:

- designing and writing content on a computer
- editing and planning with a keyboard
- working with media and visuals using a mouse
- speaking with faculty, colleagues, or learners
- using AI to speed up drafting and variation
- turning dense content into clearer learning experiences
- managing timelines, scope, and project updates

## If You Want To Change The Text

You can paste any job description into the text box.

If you want to test a different role:

1. Delete the current text.
2. Paste in the new job description.
3. Click **Analyse text** again.

## Troubleshooting

If the page opens but looks broken:

- make sure you opened `index.html`
- make sure `styles.css` and `app.js` are still in the same folder as `index.html`
- refresh the browser window

If the browser shows a blank page:

- wait a few seconds and refresh once
- try opening the file in a different browser

If you do not see any matches:

- the job description may be too short
- the wording may be very different from the sample text
- paste in a fuller description and try again
